# План разработки Telegram-бота "Приемная комиссия МПТ"

Дата плана: 2026-07-09

## Краткое резюме по исходным файлам

- [x] Из файла `ТЗ для тг бота.docx` взяты цель бота, приветствие, логика Inline-навигации, главное меню, разделы, кнопки, требование кнопок `Назад` и `Главное меню`, идея будущей админ-панели.
- [x] Из файла `ответы на вопросы для бота.docx` взята база знаний для разделов: поступление, документы, вступительные испытания, индивидуальные достижения, сроки приема, конкурс и зачисление, платное обучение, контакты, Госуслуги, частые вопросы.
- [x] Зафиксировано, что первая версия должна быть MVP без полноценной админ-панели, но с такой структурой, чтобы ее можно было добавить позже.

## Выбранный стек

- [x] Python 3.
- [x] `aiogram` для Telegram-бота.
- [x] `python-dotenv` для загрузки настроек из `.env`.
- [x] JSON-файлы для базы знаний, ссылок и карточек специальностей.

Почему так: стек простой, надежный и понятный. `aiogram` хорошо подходит для ботов с Inline-кнопками и обработкой состояний, `.env` удобно отделяет секреты от кода, а JSON легко редактировать без изменения логики бота.

## Предлагаемая структура проекта

- [x] `bot/main.py` - точка запуска бота.
- [x] `bot/config.py` - загрузка токена, ID администраторов и настроек.
- [x] `bot/handlers/start.py` - команда `/start` и главное меню.
- [x] `bot/handlers/menu.py` - обработка разделов, вопросов и ответов.
- [x] `bot/handlers/ask_question.py` - сценарий "Задать вопрос".
- [x] `bot/keyboards.py` - Inline-кнопки и навигация.
- [x] `bot/services/content.py` - загрузка базы знаний из JSON.
- [x] `data/knowledge_base.json` - вопросы и ответы.
- [x] `data/specialties.json` - список специальностей и карточки.
- [x] `data/links.json` - полезные ссылки.
- [x] `.env.example` - пример настроек.
- [x] `requirements.txt` - зависимости.
- [x] `README.md` - инструкция запуска и список того, что нужно дозаполнить.
- [x] `.gitignore` - исключения для `.env`, кэша и служебных файлов.
